
**1. У чому різниця між authentication і authorization?**

**Authentication (аутентифікація)** — це перевірка, **хто ти є**

- логін + пароль
- OTP, біометрія\
  система підтверджує особу користувача

**Authorization (авторизація)** — це перевірка, **що тобі дозволено**

- доступ до ресурсів
- ролі (admin, user)

Коротко:

- Authentication = ідентифікація
- Authorization = права доступу

**2. Чому не можна зберігати паролі відкрито? Що таке сіль?**

**Чому не можна:**

- при витоку БД всі паролі одразу стають відомі
- користувачі часто повторно використовують паролі
- це критичний ризик безпеки

**Сіль (salt)** — це випадкове значення, яке додається до пароля перед хешуванням

Приклад:

hash(password + salt)

**Навіщо:**

- навіть однакові паролі мають різні хеші
- захист від атак типу Rainbow Table
- ускладнює підбір паролів

**3. Структура JWT токена**

JWT складається з трьох частин:

Header.Payload.Signature

**Header:**

- тип токена (JWT)
- алгоритм підпису

**Payload:**

- дані (claims)
  - userId
  - role
  - exp (час життя)

**Signature:**

- підпис для перевірки цілісності

**4. Різниця між симетричним і асиметричним підписом токена**

**Симетричний підпис:**

- один і той самий ключ для підпису і перевірки
- швидший
- менш безпечний при масштабуванні

приклад: HMAC (HS256)

**Асиметричний підпис:**

- два ключі:
  - приватний (підписує)
  - публічний (перевіряє)
- безпечніше для розподілених систем

приклад: RSA (RS256)

**5. Що таке Brute Force і Rainbow Table, як захищатися?**

**Brute Force** — перебір усіх можливих паролів\
**Rainbow Table** — заздалегідь обчислені хеші паролів

**Захист:**

- хешування (bcrypt, argon2)
- використання солі
- rate limiting (обмеження спроб)
- CAPTCHA
- 2FA (двохфакторна аутентифікація)
- блокування акаунта після N спроб

**6. Що таке RBAC?**

**RBAC (Role-Based Access Control)** — модель доступу на основі ролей

Ідея:

- користувачам призначають ролі
- ролям — права доступу

**Приклад:**

- Admin → всі права
- Manager → редагування
- User → тільки перегляд

Коротко:

- RBAC спрощує управління доступом
- замість налаштування прав для кожного користувача окремо використовуються ролі
