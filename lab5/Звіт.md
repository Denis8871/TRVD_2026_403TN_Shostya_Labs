**Мета роботи:**

Реалізувати сучасну клієнтську частину системи управління задачами (SPA), забезпечити преміальний UI/UX, інтегрувати додаток із хмарним API Supabase та впровадити розширені функції фільтрації, пошуку та візуалізації прогресу.

**1. Архітектура фронтенду та технології**

Проєкт реалізований як Single Page Application (SPA) на базі **React 18** (збірка через **Vite**).

- **UI-система:** Власна дизайн-система на базі CSS-змінних (Custom Properties) для підтримки темної теми.
- **Іконки:** Бібліотека lucide-react (векторні SVG-іконки).
- **Сповіщення:** react-hot-toast (анімовані спливаючі повідомлення).
- **Маршрутизація:** react-router-dom v6.

**Структура проєкту:**

- src/components/Layout — спільні компоненти (Header, ProtectedRoute).
- src/components/Tasks — компоненти для роботи із задачами (TaskForm).
- src/pages — сторінки додатку (Dashboard, Profile, Admin, Auth).
- src/api/taskService.js — Service Layer для взаємодії з API.
- src/context/AuthContext.jsx — глобальне управління сесією.

**2. UI/UX та Дизайн-система**

У межах роботи реалізовано преміальний інтерфейс із наступними особливостями:

- **Dark Mode:** Повноцінна підтримка темної теми з плавними переходами та збереженням стану в localStorage.
- **Micro-interactions:** Анімовані кастомні чекбокси, ефекти наведення на картки задач, плавне заповнення прогрес-бару.
- **Typography:** Використано шрифт **Inter** для забезпечення кращої читабельності.
- **Toasts:** Замість стандартних alert() впроваджено систему неблокуючих сповіщень про успіх чи помилки операцій.

**3. Функціональні можливості Dashboard**

Головна сторінка (Dashboard.jsx) містить розширений інструментарій для управління даними:

- **Widget Progress:** Динамічний прогрес-бар, що в реальному часі вираховує % виконаних задач.
- **Smart Search:** Рядок миттєвого пошуку, що фільтрує задачі за назвою чи описом під час введення тексту.
- **Advanced Sorting:** Можливість сортування списку за новизною, пріоритетом (High/Medium/Low) або дедлайном.
- **CRUD Operations:** Створення та редагування задач через інтерактивну форму з валідацією обов'язкових полів.

**4. Профіль користувача та Адміністрування**

- **ProfilePage:** Реалізовано інтерфейс для персоналізації акаунта. Користувач може оновити своє повне ім'я, яке миттєво синхронізується з базою даних.
- **AdminPanel:** Захищена сторінка для управління користувачами. Доступ до неї обмежений рольовою моделлю та додатковим екраном блокування (Master Password) для підвищення безпеки.

**5. Технічна інтеграція з API**

- **SDK:** Використовується @supabase/supabase-js, який автоматично керує життєвим циклом JWT-токенів.
- **Auth Guards:** Маршрути /tasks, /admin та /profile захищені компонентом ProtectedRoute.
- **Error Handling:** Реалізовано централізовану обробку помилок мережі та бази даних із виведенням зрозумілих Toast-повідомлень.

**6. Як запустити і перевірити**

1. Клонувати репозиторій.
1. Встановити залежності: npm install.
1. Додати файл .env.local з ключами VITE\_SUPABASE\_URL та VITE\_SUPABASE\_ANON\_KEY.
1. Запустити в режимі розробки: npm run dev.
1. Відкрити http://localhost:5173.

**Висновок**

У межах ЛР №5 повністю реалізовано клієнтський застосунок на React, що відповідає сучасним стандартам веб-розробки. Додаток забезпечує високу швидкість роботи (SPA), має адаптивний дизайн, підтримує темну тему та надає користувачу зручні інструменти для організації задач (пошук, сортування, прогрес). Інтеграція з Supabase дозволила створити безпечний та функціональний продукт із мінімальними витратами на підтримку серверної інфраструктури.
