

**1. Що таке SPA і чим воно відрізняється від MPA?**

**SPA (Single Page Application)** — це додаток, який завантажується один раз і далі динамічно оновлює контент без перезавантаження сторінки.

**Особливості:**

- швидкий UX після першого завантаження
- клієнтський рендеринг
- використання API

**MPA (Multi Page Application)** — класичний підхід, де кожна дія веде до повного перезавантаження сторінки.

**Особливості:**

- рендеринг на сервері
- простіша логіка
- повільніші переходи

**Різниця:**

- SPA — одна сторінка, динаміка
- MPA — багато сторінок, повне перезавантаження

**2. Що таке Virtual DOM і навіщо він потрібен?**

**Virtual DOM** — це легка копія реального DOM у пам’яті.

**Як працює:**

1. створюється віртуальне дерево
1. при зміні стану створюється нова версія
1. відбувається порівняння (diffing)
1. оновлюються лише змінені елементи

**Навіщо:**

- підвищує продуктивність
- мінімізує прямі операції з DOM

**3. Як працює routing на клієнті?**

Клієнтський роутинг — це зміна URL без перезавантаження сторінки.

**Як це відбувається:**

- використовується History API (pushState, replaceState)
- роутер відслідковує URL
- відповідний компонент рендериться на сторінці

**Приклад:**

- /home → Home компонент
- /profile → Profile компонент

**4. Де безпечніше зберігати JWT?**

**Варіанти:**

**1. HttpOnly cookies (рекомендовано)**

- недоступні з JavaScript
- захист від XSS
- можна використовувати з Secure + SameSite

**2. LocalStorage / SessionStorage**

- прості у використанні
- але вразливі до XSS

**Висновок:**

- найкраще — HttpOnly cookies
- якщо використовуєш LocalStorage — потрібен сильний захист від XSS

**5. Що таке CORS і чому браузер блокує запити?**

**CORS (Cross-Origin Resource Sharing)** — це механізм безпеки браузера.

Він контролює запити між різними доменами (origin).

**Чому блокує:**

- щоб захистити користувача від небезпечних сайтів
- наприклад, щоб сторонній сайт не міг читати дані з іншого API

**Як працює:**

- сервер має явно дозволити доступ через заголовки:
  - Access-Control-Allow-Origin
  - Access-Control-Allow-Methods

**6. Що таке State Management?**

**State Management** — це управління станом додатку (даними, які змінюються).

**State — це:**

- дані користувача
- список товарів
- статуси (loading, error)

**Підходи:**

- локальний state (useState)
- глобальний state:
  - Redux
  - MobX
  - Zustand

**Навіщо потрібен:**

- синхронізація даних між компонентами
- передбачуваність поведінки додатку
- зручність масштабування

Коротко:

- SPA — швидкий клієнтський додаток
- Virtual DOM — оптимізація DOM
- routing — зміна сторінок без reload
- JWT краще в cookies
- CORS — захист браузера
- State Management — контроль даних у додатку
